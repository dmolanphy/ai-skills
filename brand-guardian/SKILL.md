# Brand Guardian

Build brand systems that hold together, and defend them once they exist. Brands succeed
through consistency and die through fragmentation — every deliverable either compounds
the brand or erodes it.

Strategy before artifacts, always: establish the foundation before designing logos,
picking palettes, or writing taglines. Tactical work without a foundation produces
pretty things that don't ladder to anything.

## Brand foundation framework

Work through these in order — each depends on the one before:

- **Purpose** — why the brand exists beyond profit. One sentence, no poetry.
- **Vision** — the aspirational future state it's building toward.
- **Mission** — what it does, for whom, concretely.
- **Values** — 3 core principles max, each with a behavioral manifestation ("we do X,
  which looks like Y"). Values without behaviors are wall art.
- **Personality** — 3 human traits with how each expresses in design and language.
- **Promise** — the one thing customers can always expect.

Then positioning: target audience (primary + secondary), competitive differentiation
(what only this brand can claim), 3–5 brand pillars, and a one-sentence positioning
statement. Pressure-test it: if a competitor could say the same sentence, it isn't
positioning.

Present foundation work as 2–3 distinct strategic directions with a named POV each,
not one merged compromise (David works options-first).

## Visual identity system

Specify, at minimum: logo system (primary, horizontal, stacked, icon; clear space;
minimum sizes; do's and don'ts), color system (primary / secondary / neutral palettes
with hex values and WCAG-compliant combinations flagged), typography (display + body,
hierarchy, web fallbacks), and any pattern/illustration language.

For token-level implementation (variables, Figma foundations, semantic tokens), hand
off to the `ds-foundations` skill — this skill defines the identity, that one builds it.

## Voice & messaging architecture

- Voice characteristics: 3 traits, each with usage context and an example line.
- Tone map: how voice flexes by context (sales page vs. error message vs. crisis).
- Messaging: tagline, value proposition, 3 key messages ranked by audience priority.
- Writing guidelines: preferred vocabulary, banned phrases, inclusive-language rules.

Voice work should align with the `writing-style` skill's standards: plainspoken,
no corporate jargon, confident but not performative.

## Brand audits & consistency checks

When reviewing work against an existing brand:

1. Check against foundation first (does this express the values and personality?),
   then against visual/verbal guidelines (does it follow the rules?). Flag which level
   the violation lives at — strategy misses matter more than spacing misses.
2. Distinguish drift from evolution: repeated small deviations that weaken the system
   vs. deliberate flexes that extend it. Name which one you're seeing and why.
3. Deliver findings as: what's working / what's eroding / what to fix first. Rank by
   damage, not by ease.

## Evolution calls

When asked about refreshing or extending a brand: consistency is the default; change
needs a reason rooted in audience or market shift, not boredom. A brand that can't
flex is brittle, but a brand that flexes weekly is noise. State the tension, give the
recommendation, show what stays fixed no matter what.
